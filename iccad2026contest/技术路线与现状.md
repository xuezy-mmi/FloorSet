# ICCAD 2026 FloorSet 比赛 —— 技术路线与现状总结

> 最后更新：2026-06-09
> 目标：将验证集 100 个测试用例的加权总分降到 **3 以下**。

---

## 1. 比赛与评分公式速览

- **比赛**: ICCAD 2026 CAD Contest Problem C —— FloorSet Challenge
- **数据集**: FloorSet-Lite（矩形版），21–120 个 block 的 SoC 平面规划
- **本地验证集**: 100 个测试用例；隐藏测试集由组委会保留

**单 case 成本**:
```
Cost = (1 + 0.5·(HPWL_gap + Area_gap)) · exp(2·V_rel) · max(0.7, R^0.3)
```
- 不可行 → 成本 = M = **10.0**
- 可行成本被 cap 在 `M − 1e-6 = 9.999999`，保证"任何可行 < 任何不可行"
- `HPWL_gap` / `Area_gap` 相对 baseline 不允许负值（即 beating baseline 不给奖励）
- `V_rel` 是软约束违反率，**排除 fixed/preplaced**（它们是硬约束）

**总分**:
```
Total = Σ Cost[i] · exp(n_i/12) / Σ exp(n_i/12)
```
- **大 case（n=116–120）权重约 22026**，小 case（n=21）权重仅约 5.7
- 因此**少数大 case 不可行就会把总分拉到 9.999**

## 2. 当前技术路线（两条并行 pipeline）

### 2.1 经典路线：B\*-树 + 模拟退火（SA）

**核心文件**: `dit_optimizer_v2_sa.py`

这是当前提交给评估器的方案。流程：

1. **识别 locked block**: preplaced 与 fixed 都加入 `locked` 集合
2. **以 preplaced 为根**建 B\*-树；其余 free block 用 `sqrt(area)` 初始化
3. **SA 主循环**（1.5s 时间预算）:
   - `move_rotate(b)`: 旋转单个 free block（locked / root 跳过）
   - `move_delete_insert(b)`: 删除并重新插入 free block（locked / root 跳过）
4. **后处理流水线**:
   - 把整体平移，使 preplaced 落到目标 (x, y)
   - fixed block 的 (w, h) 直接覆盖
   - bbox 原点归零
   - preplaced 的 (x, y, w, h) **显式再覆盖**一次
   - MIB 后处理：以 locked 成员为标准，传播 (w, h) 到同组其他成员
   - 第一遍 de-overlap：50 次外循环，把 free block 推出 locked block（按 overlap 较小方向）
   - 第二遍 de-overlap：贪心候选点搜索，候选点 = (0,0) ∪ 每个已放置 block 的 8 个邻接位置；按 (y, x) 排序取最小
   - bbox 再次归零
   - preplaced 再覆盖一次（补偿 bbox 移动）

### 2.2 神经网络路线：DiT（Diffusion Transformer）

**核心文件**: `dit_model_v3.py`（模型），`train_dit_v2.py` / `train_dit_v3.py`（训练）

**设计要点**:
- **排列等变性**：用 `TransformerEncoder` 做 block 之间的 self-attention，不引入破坏对称性的 pos_embed
- **图条件聚合**: 每个 block 一个 8 维条件向量
  - `log(area)`, `Σ b2b weight`, `b2b degree`, `Σ p2b weight`,
  - `p2b 加权 pin x/y`, `is_fixed_or_preplaced`, `boundary_code/15`
- **训练目标**: x₀ 参数化
  - 主损失 = 向量化比赛成本（HPWL + bbox 面积 + overlap + 面积容差 + 软约束）
  - 辅助损失 = diffusion noise MSE（小权重 `λ=0.05`）
  - EMA: 0.999 衰减
- **归一化**: per-channel z-score（w, h, x, y 各自均值/方差），修复 (w,h) 与 (x,y) 量级差异
- **推理**: 计划用 DDIM 50 步采样；硬约束在推理后端覆盖

**目前状态**: v2、v3 训练脚本已就绪，模型尚未作为最终提交主体。

## 4. 主要问题诊断

### 4.1 硬约束失败（28 个不可行 case 的主要来源）

按可能性排序：

1. **SA 超时** → 返回的 layout 不完整 / 多个 free block 互相重叠
2. **de-overlap 失效**: 50 次外循环 + 单点贪心搜索可能找不到可行位置（block 被 locked block 围死）
3. **MIB 后处理产生新 overlap**: 改 (w, h) 之后再 de-overlap 才正确；已修复（移到了 de-overlap 之前）
4. **bbox 归零把 preplaced 挪走**: 第二次 override 已经补偿

### 4.2 软约束完全没处理

代码里有 MIB 传播，但 **boundary snap** 和 **cluster abutment** **完全没实现**：
- 边界（bitmask L/R/T/B）要求的 block 没被推到 bbox 边
- 同 cluster 的 block 之间没有 abut，V_cluster 必然全错
- `exp(2·V_rel)` 一旦 `V_rel > 0.35`，成本就翻倍以上

### 4.3 SA 质量不足

- `time_budget_s = 1.5s` 太少，大 case 几乎只跑了几次外循环
- `moves_per_temp = 8` 太保守
- 没有 move_swap（两个 free block 互换 (w, h)）的路径，导致初始形状不合理时难以收敛
- `bbox_area * 0.01` 这一项权重太轻，无法驱动紧凑布局

## 5. 三档目标与对应瓶颈

| 目标 | 含义 | 主要瓶颈 |
|---|---|---|
| **第一档：避免不可行** | 把 28 个失败 case 救回可行 | de-overlap 兜底、时间预算 |
| **第二档：成本 < 3** | 在可行的前提下让 V_rel → 0，gap → 0 | boundary snap, cluster abutment, SA 质量 |
| **第三档：成本 < 1.5** | 接近最优 | HPWL/area 双 gap 接近 0，soft 全 0 |

**当前在第一档**。距离第三档需要同时满足：
- 100% 可行
- `V_rel ≈ 0`（所有 soft 全过）
- `HPWL_gap + Area_gap ≈ 0`（接近 baseline）

## 6. 下一步可执行方向（仅分析，未实施）

### 6.1 立即可做、低风险

1. **de-overlap 强化**
   - 第二遍贪心从 8 个邻接扩展到所有 block 边界的"内缩候选"
   - 用 `itertools.product` 在 (x_off, y_off) ∈ {-1, 0, 1} 网格上对每个候选微调
   - 对每个 free block 跑 1ms 的局部爬山，避开 locked 即可
2. **time_budget 动态化**: 1.5s 在 n=120 case 上可能仅够一次外循环；按 `1.5 + 0.01·n` 设定
3. **SA move_swap** 加进 move 集合（两个 free block 互换 w, h）

### 6.2 中等改动、中高收益

4. **Boundary snap 后处理**: 对 `constraints[i, 4] != 0` 的 free block，检查它当前是否在某 bbox 边/角上；不是就平移到该边/角（只动 x 或 y，保持 w/h）。需要保证平移后不引入新 overlap
5. **Cluster abutment 后处理**: 对同 cluster 的 block 用贪心拼接：先放一个根，其余依次贴上去（用与 de-overlap 相似的"邻接候选"搜索）
6. **MIB 软约束**: 已有 hard 版的 MIB 传播；软版需保证 `(w, h) ≈ 同一值` 而非强制相等（通过面积反推）

### 6.3 较大改动、可能高收益

7. **把 DiT 真正接到 SA 上**: 用 DiT 给出 4 维 (w, h, x, y) 的初始估计，喂给 B\*-树 SA 做局部精修，叠加 hard override + de-overlap
8. **DiT 输出直接做软约束解码**: 在推理时显式 `argmax` boundary mask；用 Hungarian 匹配做 cluster
